# Movies Database | SQL Portfolio Project

## О проекте

Проект демонстрирует навыки проектирования и работы с реляционной базой данных на PostgreSQL.

Реализована небольшая база данных фильмов с поддержкой связей между таблицами и аналитическими SQL-запросами.

Цель проекта — показать владение:
- созданием структуры БД
- работой с ограничениями целостности
- написанием JOIN-запросов
- использованием агрегатных функций
- применением GROUP BY и HAVING

---

## Структура базы данных

### 1. Таблица genre
Справочник жанров фильмов.

Поля:
- id — первичный ключ
- name — уникальное название жанра

### 2. Таблица movie
Основная таблица с информацией о фильмах.

Поля:
- id — первичный ключ
- title — уникальное название фильма
- rating — рейтинг (numeric)
- runtime — продолжительность в минутах
- genre_id — внешний ключ на таблицу genre

Реализованы ограничения:
- PRIMARY KEY
- FOREIGN KEY
- UNIQUE
- NOT NULL

---

## Реализованный функционал

В проекте представлены SQL-запросы, демонстрирующие:

✔ Выборку данных с фильтрацией (WHERE)  
✔ Сортировку (ORDER BY)  
✔ Соединение таблиц (JOIN)  
✔ Агрегатные функции (COUNT, AVG)  
✔ Группировку данных (GROUP BY)  
✔ Фильтрацию агрегированных данных (HAVING)  

Примеры аналитики:
- фильмы с рейтингом выше 8.5
- фильмы длительностью более 120 минут
- количество фильмов по жанрам
- средний рейтинг по жанрам
